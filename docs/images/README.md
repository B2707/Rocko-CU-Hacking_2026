# Images

Photos of the build and the event, used by the top level `README.md` and the
docs under `docs/`.

Not to be confused with `photo/` at the repo root, that folder is the injury
photo classifier (code), not pictures.

## Naming

Lowercase, hyphenated, descriptive. The filename should say what the picture
shows, so a link in the README still makes sense a year from now.

## Current inventory

```
rig.jpg                  the bench rig: coil, Pi, live dashboard (used in README)
team.jpg                 the team at CU Hacking 2026 (used in README)
building-rocko.jpg       mid-build at the bench, listener log on screen (used in README)
awards.jpg               the awards ceremony (used in README)
prize-table.jpg          candid at the prize table (spare, not embedded)
```

All four are resized to 1600 px and re-encoded (EXIF checked, no GPS data).
Hardware close-ups (the coil, the L298N wiring, the Pico front end) would be
a good addition if photos exist.

## Format

JPEG for photos, PNG for screenshots and diagrams. Keep each file under about
1 MB, these are read in a browser on GitHub, not printed. Resize anything
straight off a phone camera before committing it.
